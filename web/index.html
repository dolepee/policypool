Pool rules, enforced before the swap.

PolicyPool lets a v4 pool publish the max-swap and daily-volume covenant its liquidity will accept. Same trader, same exact-input swap, different pool covenant: one executes, one refuses.

Hook mechanism

One callback. One covenant check. No offchain matcher.

PolicyPool uses `beforeSwap` only. The pool policy is loaded by `PoolId`, checked before execution, and updated in the same swap path. The demo router only catches the refused strict-pool swap so indexers can see the blocked outcome cleanly. 

if amountIn > maxSwapAmount:
  revert PolicyBlocked("MAX_SWAP_EXCEEDED")

if spentToday + amountIn > dailyCap:
  revert PolicyBlocked("DAILY_CAP_EXCEEDED")

spentToday += amountIn
emit SwapAccepted(poolId, trader, amountIn)
