Uniswap v4 Hook · X Layer mainnet

Liquidity with a veto.

PolicyPool lets each v4 pool publish the max-swap and daily-volume limits its liquidity will accept. The Hook enforces both limits inside beforeSwap, before the trade touches the pool.

Hook mechanism

A covenant check in the swap path.

The first version is deliberately narrow: one Hook, one callback, two pool-level limits. No oracle, no offchain matcher, no governance layer. 

if amountIn > maxSwapAmount:
  revert PolicyBlocked("MAX_SWAP_EXCEEDED")

if spentToday + amountIn > dailyCap:
  revert PolicyBlocked("DAILY_CAP_EXCEEDED")

spentToday += amountIn
emit SwapAccepted(poolId, trader, amountIn)
